# Team Standards

> Development standards for GlobalFlavors project

---

## Commit Messages

Format: `type: what you did`

**Types:**
- `feat` - New feature
- `fix` - Bug fix
- `docs` - Documentation
- `refactor` - Code restructuring
- `test` - Tests
- `chore` - Maintenance

**Examples:**
```
feat: add recipe search filter
fix: resolve login timeout issue
docs: update setup instructions
```

---

## Pull Requests

Before merging, verify:
- [ ] Code works as expected
- [ ] Tests pass (if applicable)
- [ ] Documentation updated (if needed)
- [ ] Someone reviewed it

**PR Title:** Same format as commits (`type: description`)

---

## Documentation

- Keep docs simple and scannable
- Update when making related code changes
- Use team attribution, not individual authors
